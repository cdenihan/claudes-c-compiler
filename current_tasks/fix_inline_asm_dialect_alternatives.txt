Fix inline assembly GCC dialect alternatives syntax ({att|intel})

GCC's inline assembly supports dialect alternatives: {att_syntax|intel_syntax}
where the first alternative is used for AT&T syntax and the second for Intel syntax.

Example from cpuid.h:
  __asm__("pushf{l|d}\n\t"
          "pop{l}\t%0\n\t"
          "mov{l}\t{%0, %1|%1, %0}\n\t"
          "xor{l}\t{%2, %0|%0, %2}\n\t")

Since our compiler always uses AT&T syntax, we take the first (ATT) alternative
from each {a|b} group:
  pushf{l|d} -> pushfl
  mov{l}\t{%0, %1|%1, %0} -> mov{l}\t%0, %1 -> movl\t%0, %1

This affects projects using __get_cpuid() from GCC's cpuid.h on i686 targets
(32-bit x86), where the header uses dialect alternatives for CPUID detection.

Affected projects: sophia (i686), and any project using <cpuid.h>
