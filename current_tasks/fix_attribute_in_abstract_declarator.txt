Fix: __attribute__ inside abstract declarators in cast expressions

Bug: The parser fails to parse cast expressions like `(int(__attribute__((noinline)) *)(void))`.
The issue is in `try_parse_paren_abstract_declarator()` in declarators.rs which doesn't skip
__attribute__ tokens before the pointer '*' inside parenthesized abstract declarators.

This causes test c_testsuite_00210 to fail with "unexpected token Int in expression".

Fix: Add attribute skipping in try_parse_paren_abstract_declarator() before the star-consuming loop.
