Fix macro parameter substitution: sort by longest name first to avoid prefix collisions.

When a macro has parameters like `orig`, `alt`, `orig_len`, `alt_len`, the
substitution of `\orig_len` incorrectly matches `\orig` first (since it's a
prefix), leaving `_len` as literal text. This produces bogus symbol references
like `140b_len` and `143f_len` in the Linux kernel's ALTERNATIVE macros.

The ARM assembler parser already has the fix (sort params by name length,
longest first). Apply the same fix to:
- src/backend/asm_preprocess.rs (expand_macros and expand_macros_with)
- src/backend/x86/assembler/parser.rs (expand_gas_macros_with_state)
