Fix va_arg with complex type default argument promotions

Problem:
When float _Complex is passed to a variadic function, the C standard requires
default argument promotion to double _Complex. Similarly, va_arg(ap, float _Complex)
must read a double _Complex from the va_list and convert back to float _Complex.

Currently:
1. Caller side: Only F32->F64 promotion exists, not float _Complex -> double _Complex
2. va_arg side: lower_va_arg_complex reads component type directly without promotion

Fix:
1. Add float _Complex -> double _Complex promotion in variadic call argument lowering
2. Fix va_arg to read promoted type and convert back for float _Complex
3. Handle x86-64 packed complex float convention properly in promotion context

Affected tests: compiler_suite_0172_0038, compiler_suite_0081_0002,
compiler_suite_0109_0089, and other va_arg complex failures
