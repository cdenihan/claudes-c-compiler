Task: Fix ## token paste operator absorbing non-parameter literal identifiers
Status: in_progress

Bug: In macros like `#define M(_name, _div) _name##_##div##_div`,
the preprocessor's ## handling would greedily absorb literal tokens
(`_` and `div`) into the next ##'s left operand extraction, accidentally
forming a parameter name (`_div`) and substituting it. This caused
`M(foo, 2)` to produce `foo22` instead of the correct `foo_div2`.

Fix: Wrap non-parameter literal right-operand identifiers in paste-protection
markers (\x02/\x03) so extract_trailing_ident can't greedily absorb them.

This fixes ARM kernel build failures in drivers/clk/qcom/clk-rpmh.c which
uses the DEFINE_CLK_RPMH_ARC macro with this exact pattern.
