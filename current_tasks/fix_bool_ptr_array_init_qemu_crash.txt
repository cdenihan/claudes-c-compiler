Fix: bool pointer array initialization incorrectly normalizes pointers to booleans

When initializing a local array of `bool*` (pointers to bool), the compiler
incorrectly applies bool normalization to the pointer values, converting them
to 0/1 instead of storing the actual addresses.

This causes a segfault in QEMU's migrate_mark_all_params_present() which
uses `bool *has_fields[] = { &p->has_X, ... }` and then dereferences the
stored pointers.

Fix: Guard the is_bool_elem_array check in lower_1d_array_init to exclude
arrays of pointers (is_array_of_pointers, is_array_of_func_ptrs).

Status: in_progress
