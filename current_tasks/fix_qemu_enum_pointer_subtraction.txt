Fix QEMU enum pointer subtraction type errors

QEMU uses VMSTATE_UINT32 macros that do compile-time type assertions via
(uint32_t*)0 - (typeof(field)*)0. When the field is an enum type, our
compiler rejects this as "invalid operands to binary -" because
pointee_types_compatible() doesn't recognize enums as compatible with
integer types.

This causes 6 compilation failures in QEMU x86_64-softmmu:
  - hw/audio/sb16.c (AudioFormat enum)
  - hw/tpm/tpm_tis_common.c (anonymous enum)
  - hw/usb/hcd-xhci.c (TRBType enum)
  - hw/usb/dev-storage.c (USBMSDMode enum)
  - hw/vfio/cpr.c (anonymous enum)
  - hw/display/virtio-gpu.c (anonymous enum)

Fix: Make pointee_types_compatible() in sema/analysis.rs treat enum
types as compatible with integer types for pointer subtraction.
