Task: Implement -mcmodel=kernel support for x86 codegen
Status: in_progress

Problem: The compiler always uses RIP-relative addressing (leaq symbol(%rip), %rax)
for global symbol address loads, but with -mcmodel=kernel, GCC uses absolute
32-bit sign-extended addressing (movq $symbol, %rax with R_X86_64_32S).

This matters for very early boot code like __startup_64 in head64.c, which runs
at physical addresses different from the linked virtual addresses. RIP-relative
references resolve to wrong addresses in that context.

Fix:
1. Parse -mcmodel=kernel flag and store in CodegenOptions
2. When mcmodel=kernel, emit "movq $symbol, %rax" instead of "leaq symbol(%rip), %rax"
   for global address loads in emit_global_addr()
3. This produces R_X86_64_32S relocations instead of R_X86_64_PC32

Files to modify:
- src/main.rs: parse -mcmodel=kernel
- src/backend/mod.rs: add code_model field to CodegenOptions
- src/backend/state.rs: propagate code_model to CodegenState
- src/backend/x86/codegen/codegen.rs: modify emit_global_addr()
