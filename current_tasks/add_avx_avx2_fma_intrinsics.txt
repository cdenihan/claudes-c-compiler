Task: Add AVX, AVX2, and FMA3 intrinsic header support
Status: in_progress

Goal: Enable OpenBLAS HASWELL target to compile by adding:
1. avxintrin.h - __m256, __m256d types and AVX intrinsics (load/store/arithmetic/shuffle/etc)
2. avx2intrin.h - __m256i type and AVX2 integer intrinsics
3. fmaintrin.h - FMA3 intrinsics (_mm256_fmadd_ps, etc)
4. _MM_TRANSPOSE4_PS macro in xmmintrin.h
5. Update immintrin.h to include all new headers
6. Fix openblas check scripts (missing test_openblas.c)

Approach: Follow existing pattern of implementing SIMD types as aligned C structs
and intrinsics as inline C functions (no new compiler builtins needed for pure
arithmetic/memory operations).
